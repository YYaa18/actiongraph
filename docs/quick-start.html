Golden PathL0 · Schema-first Goal（15 分钟）

先只掌握 3 个概念：Goal schema 是目标参数契约，Action 是可执行业务能力，Condition 是规划器使用的符号事实。

// build.gradle.kts
dependencies {
  implementation(platform("com.actiongraph:actiongraph-bom:0.2.0-SNAPSHOT"))
  implementation("com.actiongraph:actiongraph-spring-boot-starter")
}
@Component
class HelloAgent {
  @ActionGraphGoal(type = "hello.finish",
      targetConditions = "hello:DONE",
      seedConditions = "hello:INPUT_PRESENT",
      schema = InputId.class)
  void goal() {}

  @ActionGraphAction(id = "hello.finish",
      preconditions = "hello:INPUT_PRESENT",
      effects = "hello:DONE")
  void finish(InputId input) {}
}

record InputId(
    @GoalParameter(name = "id", description = "业务输入编号", example = "A-1")
    String value
) {}
@Service
class HelloRunner {
  private final ActionGraph actionGraph;

  HelloRunner(ActionGraph actionGraph) {
    this.actionGraph = actionGraph;
  }

  RunResult run() {
    return actionGraph.start("hello.finish", Map.of("id", "A-1"));
  }
}

检查点：不需要编写 Seeder，InputId 会自动进入 Blackboard；run().status() 返回 COMPLETED。

Golden PathL1 · 加上门禁、人审和恢复

当 Action 需要看运行期对象值才能判断能否执行，用 @ActionGraphGuard；当动作有副作用，用 @ActionGraphCompensation；当动作需要人工审批，声明 requiresHumanReview = true。

@ActionGraphAction(
    id = "order.approval.request",
    preconditions = "order:DRAFT_CREATED",
    effects = "order:APPROVAL_REQUESTED",
    riskLevel = ActionRiskLevel.HIGH,
    requiresHumanReview = true)
ApprovalRequest requestApproval(OrderDraft draft) {
  return approvalService.request(draft);
}

RunResult suspended = actionGraph.start("order.cancel", params);
// 审批系统写入决定后，业务或回调处理器继续同一个 run
RunResult resumed = actionGraph.resume(suspended.runId());

检查点：第一次运行返回 SUSPENDED_PENDING_REVIEW，审批通过后 resume(...) 返回 COMPLETED。

Golden PathL2 · 自然语言入口

提供一个 GoalInterpreter Bean 后，根门面就能使用 chat(...)。缺参数不是异常，而是对话分支。

ChatResult missing = actionGraph.chat("帮这个客户准备续约报价");
missing.started();                 // false
missing.clarification().text();    // 追问 customerId

ChatResult started = actionGraph.chat(
    "帮这个客户准备续约报价",
    Map.of("customerId", "C001"));
started.run().status();            // COMPLETED 或挂起状态

LLM provider、prompt 渲染和结构化输出解析可以使用 actiongraph-llm-deepseek，也可以先用规则解释器兜底。

ProductionL3 · 生产化开关

生产化优先通过配置开启，不改变 L0/L1 的业务代码。配置建议按属性域维护，不把整份环境配置复制进业务文档。

actiongraph:
  goals:
    auto-register-annotated: true
  seeding:
    auto: true
  persistence:
    jdbc:
      enabled: true
      suspended-run-claim-timeout: 15m
      blackboard:
        allowed-packages:
          - com.example.business
  masking:
    enabled: true
  validation:
    mode: fail

检查点：schema 自动绑定保持开启；进程重启后仍可恢复挂起运行；重复 resume 只会有一次业务副作用；Trace 链可验证。

ProductionL4 · 跨服务与外部事件

当某一步要等待 MQ、回调或批处理结果，Action 返回 waiting 状态，运行会挂起到事件到达或超时。

return ActionResult.waiting(
    "payment-cleared",
    EventWaitOptions.builder()
        .correlationId(paymentId)
        .idempotencyKey(requestId)
        .timeout(Duration.ofHours(2))
        .build());

检查点：事件回调投递后恢复同一 run；重复投递依赖幂等键，不产生重复副作用。
