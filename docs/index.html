ActionGraph

面向 Java 企业系统的类型化 GOAP Agent 运行时。

把自然语言目标转成安全、可审计、可补偿的业务 Action 执行链。LLM 负责理解目标，确定性规划器负责组合路径，业务系统拥有最终裁决权。

企业 AI 执行的难点，不是回答，而是敢不敢执行。

ActionGraph 的设计目标是把“可控业务能力”变成 Agent 可以规划和执行的 Action 集合，同时把计划、审批、补偿、恢复和审计放在框架层统一处理。

在企业系统里，让 LLM 直接调 API 或写 SQL 通常不可控、不可复现，也很难审计。只做问答机器人又无法把流程办完。每个业务系统各自接 AI，则会重复建设 prompt、审批、审计和回滚机制。

ActionGraph 选择了一条更保守也更工程化的路线：业务系统只暴露可控 Action，规划器只看条件与效果，运行时每一步都经过权限、人审、runtime guard 和补偿边界。LLM 可以解释目标，但不能直接操作你的业务状态。

LLM 直连 API计划不可复现，越权和重复副作用难以治理。
只有问答机器人能解释制度，不能稳定完成审批、下单、生成草稿等流程。
系统各自接 AIprompt、审计、回滚、人审和运维标准分散，长期成本高。

从自然语言目标到可审计执行业务链。

运行时把目标解释、Blackboard 装配、确定性规划、逐步执行、人审挂起、恢复和补偿串成一个闭环。每一步执行后都会重新规划，因此运行时事实变化可以自动改道。

1

解释目标

GoalInterpreter 把用户输入映射为 GoalType 与参数。LLM 解释器可用，规则解释器可降级。

2

装配上下文

GoalBlackboardSeeder 把 customerId、orderId 等业务参数写入类型化 Blackboard。

3

确定性规划

GoapPlanner 只根据 Action 的 precondition 与 effect 搜索可达路径，不让 LLM 生成计划。

4

受控执行

GoapExecutor 按步骤执行，检查权限、runtime guard、金额规则与人审策略。

5

恢复与审计

Trace、挂起快照、审批任务、补偿栈与 Console read model 形成可追踪证据链。

快速开始：引入 starter，把普通方法变成 Action。

Spring Boot 应用通常只需要 BOM、主 starter 和一个可选 LLM provider。端点、JDBC、人审和治理都通过 actiongraph.* 属性显式启用。

build.gradle.kts 
dependencies {
  implementation(platform("com.actiongraph:actiongraph-bom:0.1.0"))

  implementation("com.actiongraph:actiongraph-spring-boot-starter")
  implementation("com.actiongraph:actiongraph-llm-deepseek")
}

Action adapter 
@Service
class OrderCancellationWorkflow {
  @ActionGraphAction(
    id = "order.lookup",
    preconditions = "order-cancellation:ORDER_ID_PRESENT",
    effects = "order-cancellation:ORDER_LOADED",
    riskLevel = ActionRiskLevel.READ_ONLY
  )
  OrderRecord lookup(OrderId orderId) {
    return orderService.lookup(orderId);
  }

  @ActionGraphGuard(actionId = "order.cancellation.request.draft")
  boolean canDraft(CancellationEligibility eligibility) {
    return eligibility.allowed();
  }

  @ActionGraphCompensation(actionId = "order.cancellation.request.draft")
  void voidDraft(CancellationRequestDraft draft) {
    cancellationRequestService.voidDraft(draft.requestId());
  }
}

主 starter 不会替你创建业务 Action 或 LLM client。

它提供规划器、执行器、仓储默认实现、注解扫描、治理策略和端点自动装配。业务动作、领域 Goal、解释器、DataSource、LLM provider 与生产权限边界仍由接入应用明确提供。

核心能力围绕一个原则：业务系统拥有最终裁决权。

框架让 Agent 能执行，但不把业务判断交给模型。规划、运行时校验、审批、补偿、持久化、审计与控制面都围绕“可控执行”设计。

确定性 GOAP 规划

只基于 Action 前置条件与效果搜索路径，计划可复现、可测试、可审计。

类型化 Blackboard

运行数据按 Java 类型与 BlackboardKey 管理，支持同类型多实例和注解参数绑定。

Runtime Guard

依赖真实业务值的判断放到执行前校验，保持规划器纯净，不污染 symbolic planning。

多级人工审批

Repository-backed review 支持审批链、stage index、回调幂等与原子 suspend/resume。

Saga 补偿

失败或审批拒绝时按已执行 Action 逆序补偿，降低孤儿副作用残留。

Trace 哈希链

TraceEvent 按 runId 形成链式 SHA-256 哈希，Console 可显示审计链完整性。

敏感数据脱敏

Trace 与审批预览可脱敏身份证、银行卡、手机号、邮箱和指定字段。

额度与风险策略

业务方提供 AmountExtractor 后，可按 action / 币种设置硬拒绝，并在进入人审时追加金额升级审批阶段。

结构化记忆

MemoryRepository 与 MemoryContextLoader 进入 core，Spring 与 JDBC 可自动切换。

模块化生态：从核心运行时到控制面端点。

每个模块都有清晰边界。Spring 服务可以用主 starter 快速收敛能力，非 Spring 和 Java 8 老系统也可以只引入控制面或 HTTP client。

runtime

actiongraph-core

Action、Planner、Executor、Policy、Trace、GoalCatalog、Runtime API service、Memory API 与 JSpecify 空安全合同。

spring

spring-boot-starter

注解扫描、默认 Bean、按配置启用的 JDBC 仓储、治理、人审、Runtime/Catalog/Review/Console 端点。

java8-client

control-plane-api

Java 8 组件目录、控制层响应契约、JSpecify 空安全合同、聚合 HTTP client、split clients 与 token verifier。

ops

actiongraph-console

只读运行查询、JDBC read model、Trace 详情、审计链验证、CSV/JSONL 导出。

review

human-review

审批任务仓储、审批链、回调处理、task API service 和 repository-backed policy。

governance

governance

脱敏、额度规则、权限策略、review attributes 与风险审批链路由。

persistence

persistence-jdbc

Trace、SuspendedRun、Trace read model、Memory 与 HumanReview 的 JDBC 仓储。

llm

llm-deepseek

Provider-neutral 目标解释、GoalCatalog prompt、结构化输出解析与 DeepSeek 兼容客户端。

推荐组合 适用场景 典型模块
core-runtime 最小纯 Java 运行内核，需要完全手工装配。 actiongraph-core
spring-business-runtime Spring Boot 业务服务接入 ActionGraph，但不公开控制面端点。 actiongraph-spring-boot-starter
durable-spring-runtime 生产运行态需要 JDBC Trace、挂起、Memory 和审批任务。 spring-boot-starter + DataSource + JDBC switch
full-control-plane 单个服务同时拥有 Runtime、Component Catalog、Human Review 与 Console 端点。 spring-boot-starter + endpoint enabled properties
java8-legacy-client Java 8 老系统通过 HTTP、网关、ESB 或 sidecar 调用部署好的 ActionGraph 服务。 actiongraph-control-plane-api

控制面默认关闭，按部署边界显式打开。

主 starter 收敛端点自动装配，但每个端点族都有自己的 path、token header、shared-secret 与 enabled 开关。启用端点不会创建业务 Action、LLM client 或领域解释器。

Runtime API

解释、启动、恢复运行。metadata overload 会把白名单请求头写入 Trace，并在高风险挂起时合并到审批 attributes。

POST/actiongraph/runtime/interpret
POST/actiongraph/runtime/runs
POST/actiongraph/runtime/runs/{runId}/resume

Component Catalog

机器可读模块目录、能力标签、依赖提示、兼容性标签和推荐组合方案，适合部署检查与网关治理。

GET/actiongraph/components
GET/actiongraph/components/modules/{module}
GET/actiongraph/components/compatibility/{compatibility}

Human Review

审批任务查询、详情、决策写入与外部系统回调。stage index 防止重复回调误推进下一阶段。

GET/actiongraph/human-review/tasks/pending
POST/actiongraph/human-review/tasks/runs/{runId}/actions/{actionId}/decision
POST/actiongraph/human-review/callbacks

Console

只读运行列表、筛选、Trace 时间线、审计链验证、CSV/JSONL 导出。Console 不执行、恢复、审批、拒绝或补偿运行。

GET/actiongraph/console/runs
GET/actiongraph/console/runs/{runId}/trace
GET/actiongraph/console/runs/{runId}/trace/export.jsonl

治理能力不是旁路插件，而是运行路径的一部分。

权限、脱敏、额度、人审、补偿、哈希链和恢复快照全部围绕同一个 runId 与 Trace 序列展开，方便企业控制台和审计系统追踪。

权限与额度

PermissionPolicy 与 ExecutionPolicyGuard 在执行前判定动作是否允许；金额规则可硬拒绝，并为进入人审的动作追加金额升级属性。

  • 按 action id 与 currency 配置 hard-limit / review-limit
  • AmountExtractor 由业务侧提供，不让框架猜业务金额
  • RiskBasedChainResolver 根据 amountEscalated 追加 amount-authorization 阶段
  • 规则权限与租户校验可在非 Spring 服务手工装配

人审挂起与恢复

requiresHumanReview 的 Action 创建审批任务并保存挂起快照，审批后从同一个 runId 原子 claim 恢复，避免重复副作用。

  • PENDING 只挂起，不补偿
  • APPROVED 推进阶段，最终阶段通过后继续执行
  • DENIED 终止并触发已执行 Action 的逆序补偿

审计与脱敏

Trace 事件先经过 DataMaskingPolicy，再写入哈希链。挂起快照保持无损恢复，不做脱敏。

  • Trace detail/data 与审批预览支持字段与正则脱敏
  • TraceChainVerifier 可识别断链与篡改
  • Console read model 可分页筛选并导出归档证据

生产接入关心的边界，框架已经给出默认答案。

JDBC 持久化、Java 8 老系统、控制层 token、GET 重试、Blackboard 类型白名单和 snapshot version 都是发布契约的一部分。

JDBC persistence 启用 actiongraph.persistence.jdbc.enabled=true 后，Trace、SuspendedRun、Trace read model、Memory 和 Human Review repository 可切到 JDBC。
Blackboard allowlist 生产恢复挂起快照时建议配置 allowed-packages，在反序列化前拒绝不允许的类型。
Java 8 control plane actiongraph-control-plane-api 编译为 Java 8，无 Spring、JDBC、LLM、Servlet 或 JSON 依赖。
Safe GET retries 可对组件目录、人审查询和 Console 只读查询开启 GET 重试；会产生副作用的 POST 不自动重试。
控制面 shared-secret 不是企业 IAM。

内置 token verifier 负责 Header 名校验、未配置 shared-secret 时跳过读取、常量时间比较 token。生产身份认证、RBAC、租户权限、网络边界和审计策略仍应由接入系统、API 网关或企业安全平台负责。
