内部技术框架提案

ActionGraph

让公司每个 Java 业务系统，安全地获得 AI 能力

用户说一句话，系统自动规划并执行业务流程 —— 但每一步都在白名单内、高风险必须人工审批、全程可审计、失败自动回滚。

技术中台 · 2026 年 6 月

为什么需要它

AI 落地业务系统，今天只有三条路 —— 每条都有致命伤

🎲

让 LLM 直接操作系统

LLM 自己写 SQL、自己调 API。不可控、不可审计，一次幻觉就可能改错正式数据 —— 没有企业敢上生产。

💬

只做问答机器人

能聊天、能查资料，但不能替用户把事办完。业务价值停留在客服层，核心流程没有提效。

🧩

各系统各自接 AI

每个团队重复造轮子：各写一套 prompt、各做一套审批和审计。成本 × N，风险敞口 × N，标准无法统一。

我们需要的是：一个统一的、安全边界内置的 AI 执行底座 —— 各业务系统按同一套标准接入。

ActionGraph 是什么

分工明确：AI 负责理解，算法负责规划，业务系统拥有最终裁决权

准备续约报价"
→
🤖 LLM 理解层识别意图 · 补全参数
缺信息就追问

⚙️ 确定性规划器在已注册的业务能力图谱中
搜索执行路径（非 LLM）
→
🛡 受控执行器权限闸门 · 人工审批
失败自动回滚
→
🏢 业务系统真正的校验与变更
规则一行不动

LLM 永远碰不到数据库

它只能从注册过的业务 Action 白名单里"选"，不能自由调 API、写 SQL、跳过审批。

执行路径是算出来的

规划由确定性图搜索完成，同样输入永远同样路径 —— 可复现、可测试、可解释。

业务逻辑零迁移

现有 Service 原样保留，只需加注解封装成 Action。业务规则的裁决权不转移。

用户视角的实际效果

从「点八个页面」到「说一句话」

好的，我将执行以下步骤：

⚠️ 不会直接发送给客户，不会修改正式合同。是否继续？
继续
✅ 报价草稿 QUOTE-1 已生成，已提交销售主管审批（审批单 APPROVAL-1）。全过程 29 条审计记录可查。

先解释，再执行

执行前向用户展示完整计划和边界声明，用户确认后才动手。

中途有变自动改道

查不到现有合同？规划器自动改走「创建合同草稿」路径，无需人写 if-else 流程。

高风险动作自动卡审批

提交审批环节自动挂起，审批人在现有审批系统里通过后继续执行。

领导最关心的问题

五道内置防线，每一道都有代码和测试背书

① Action 白名单

AI 只能调用注册过的业务能力，禁止自由 SQL / API / 脚本。能力目录由各业务系统自己定义和维护。

② 高风险必须人工审批

付款、退款、删数据、对外发送、改正式状态 —— 执行前自动挂起，审批通过才继续，拒绝则自动回滚已做动作。

③ 先解释后执行

执行计划先以人话呈现给用户确认。用户看到的，就是系统将做的 —— 不多不少。

④ 失败自动补偿（Saga）

流程做到一半失败，已完成的动作按逆序自动撤销：撤回审批、作废草稿。即使跨系统重启也不丢补偿链。

⑤ 全链路审计

用户原话 → AI 解析结果 → 生成的计划 → 每步输入输出 → 权限判定 → 审批人与意见 → 最终结果，全部入库可回放。合规审查随时拉得出完整证据链。

  • 给已有 Service 方法加 @ActionGraphAction 注解 —— 声明前置条件、效果、风险等级
  • 注册业务目标（一个目标 ≈ 10 行代码）
  • 处理 6 种运行结果状态

建议的下一步

三周试点计划：用一个真实场景验证全链路

第 1 周

选型与接入

选 1 个低风险高频场景（建议：草稿生成类 / 申请发起类），业务团队在框架组陪跑下完成 Action 封装与目标注册。

第 2 周

对接与联调

对接现有审批系统与数据库持久化，跑通「挂起 → 审批 → 恢复」与失败回滚演练，安全团队评审审计链。

第 3 周

灰度与评估

小范围真实用户灰度，采集指标，输出试点报告与推广决策建议。

成功指标（试点报告交付）

  • 单流程操作耗时下降幅度（目标 ≥ 50%）
  • 接入人天实测（目标 ≤ 3 人天）
  • 审批拦截率 100%、回滚成功率 100%、审计链完整率 100%

需要的支持

  • 1 个试点业务团队（约 0.5 人 × 3 周）
  • 框架组 1 人陪跑
  • 测试环境 LLM API 额度（成本极低，仅意图理解，单次调用 < 1 分钱）
为什么是现在

把 AI 能力变成基础设施，而不是各团队的副业

每个业务系统注册自己的能力图谱，全公司共享一个安全、可审计的 AI 执行底座。
接入的系统越多，能力图谱越大，AI 能替员工完成的事就越多 —— 这是平台效应，先建者定标准。

近期

试点落地 → 2~3 个系统接入 → 内部 Maven 发布 + 接入文档（已就绪）

中期

对接统一审批中心与权限体系，跨系统目标编排（一个目标调用多个系统的 Action）

远期

能力图谱市场：各团队发布 / 复用业务能力，新业务流程由图谱自动组合而成
