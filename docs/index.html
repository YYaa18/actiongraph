Governed Action Graph Runtime

ActionGraph

让大模型解释目标，让 Java 运行时掌控业务执行。

ActionGraph 把 Goal、Action、Condition、Policy、Trace 组织成一个可治理的 Agent 执行边界。模型不能直接生成计划、调用数据库或绕过审批；业务系统通过类型化 Action 图保留最终控制权。

v0.2.0-SNAPSHOT Java 21 Runtime Java 8 Control-plane Client Spring Boot Starter Apache-2.0
目标契约 Goal schema、配置化 Goal、Studio bundle 都进入同一个 GoalCatalog。
执行契约 确定性 GOAP planner 只看 condition/effect，每一步之后重新规划。
治理契约 RunPrincipal、Policy、Human Review、Compensation 和 Trace 全链路贯通。
运营契约 JDBC 恢复、事件等待、Console、OTel 和解释评测服务生产治理。

它不是把 LLM 接到业务系统上，而是给 Agent 执行权建立边界。

ActionGraph 的核心定位是企业 Agent 执行底座。模型只产出目标类型和参数，运行时负责计划、校验、审批、恢复和审计；业务 Action 仍由 Java 服务显式声明。

不是 LLM Plan Executor

模型不决定执行步骤

LLM 可以把自然语言解释为 GoalType 与参数，但不会生成 Action 顺序。执行路径由 GoapPlanner 根据 precondition/effect 搜索，结果可复现、可测试、可审计。

不是流程图复刻

每一步后都能改道

运行态事实会写入 Blackboard，Action 执行后重新规划。核心系统失败、策略变化、外部事件到达时，不需要把所有分支提前写死成流程。

不是控制台附属功能

运行时先有契约，再有界面

Console、Component Catalog、Goal Studio、HTTP clients 都是控制面；应用代码优先依赖 ActionGraph 根门面和明确的 runtime contracts。

模型输出GoalType、参数、缺参澄清；不输出计划。
框架负责绑定 schema、规划路径、执行 Action、管理挂起与恢复。
业务负责领域 Action、权限语义、金额提取、实体映射和最终裁决。
运营负责端点暴露、IAM、审计归档、评测门禁和生产观测。

金路径不是一个样例，而是一条可扩展的采用路线。

从两注解接入到生产评测，ActionGraph 的能力按层开放。团队可以先让一个业务 Goal 跑通，再逐步加入身份、审批、持久化、控制面和解释质量治理。

01

声明目标与业务能力

用 @ActionGraphGoal(schema=...) 定义目标参数契约，用 @ActionGraphAction 把已有 Service 方法暴露为可规划 Action。

  • 核心 API：@GoalParameter、@GoalSchema、description
  • 结果：GoalCatalog + ActionRegistry
02

让框架完成常规装配

schema 自动 seeding 默认开启，参数按 record 绑定并写入 Blackboard。复杂引用解析使用 TypedGoalValueConverter 或显式 Seeder。

  • 核心配置：actiongraph.seeding.auto
  • 逃生口：@BindGoalParams、SeedResult
03

从根门面发起运行

业务代码调用 ActionGraph.start、chat、resume。需要身份时传入 RunPrincipal，该身份会进入策略、人审任务、挂起快照和 Trace。

  • 入口：start / chat / resume
  • 身份：subject、clientId、delegationChain、roles
04

把治理放进行动前

Runtime Guard、PermissionPolicy、额度规则、人审链和补偿都在副作用发生前进入执行路径。拒绝、失败和未知结果都有明确状态。

  • 状态：SUSPENDED_PENDING_REVIEW、DENIED_BY_POLICY
  • 保证：原子 claim resume，逆序 compensation
05

进入生产运行形态

JDBC 持久化 Trace、挂起快照、Memory、人审任务和 Console read model；外部事件通过 ActionResult.waiting(...) 恢复同一个 run。

  • 生产域：durability、events、persistence.jdbc
  • 证据：Trace hash chain、CSV/JSONL export
06

把解释质量纳入工程门禁

Goal Studio 只在非生产环境起草 Goal bundle，解释器用 JSONL golden set 和生产脱敏采样持续评测，观测输出可进入 Micrometer 或 OpenTelemetry。

  • 门禁：Goal fingerprint、Action graph fingerprint
  • 评测：goalType、parameters、clarification accuracy

运行模型保持简单：解释目标，绑定事实，规划路径，受控执行。

ActionGraph 把不稳定的自然语言入口和可审计的业务执行分开。每一次运行都有 runId、principal、fingerprints、Trace 和明确的终态或挂起态。

1
Interpret

GoalInterpreter 输出 GoalInterpretation；缺参时返回 clarification，不启动运行。

2
Seed

Goal schema record 或显式 Seeder 写入 Blackboard，并添加 seed conditions。

3
Plan

GoapPlanner 基于当前 condition/effect 搜索可达路径，不读取模型生成的步骤。

4
Execute

每个 Action 执行前检查权限、runtime guard、人审、retry、timeout 和 policy。

5
Suspend / Resume

人审和外部事件暂停同一个 run；审批或事件到达后原子认领并恢复。

6
Evidence

Trace、fingerprints、hash chain、Console read model 和评测采样形成证据面。

Schema-first Goal + Governed Action 
@ActionGraphGoal(
    type = "renewal.quote",
    description = "Prepare a renewal quote and request approval.",
    targetConditions = "renewal:APPROVAL_REQUESTED",
    seedConditions = "renewal:CUSTOMER_ID_PRESENT",
    schema = RenewalRequest.class)
void renewalQuoteGoal() {}

record RenewalRequest(
    @GoalParameter(name = "customerId", example = "C001")
    String customerId
) {}

@ActionGraphAction(
    id = "renewal.customer.load",
    description = "Load customer profile for renewal.",
    preconditions = "renewal:CUSTOMER_ID_PRESENT",
    effects = "renewal:CUSTOMER_LOADED")
CustomerProfile load(RenewalRequest request) {
    return customerService.load(request.customerId());
}

RunPrincipal principal = new RunPrincipal(
    "sales-user-42", "crm-gateway", List.of(), Map.of("roles", "sales"));

RunResult result = actionGraph.start(
    "renewal.quote", Map.of("customerId", "C001"), principal);

治理不是运行之后补日志，而是在 Action 执行前做裁决。

ActionGraph 将身份、权限、金额、人审、补偿、外部事件和审计链路放到同一个运行模型里，避免每个业务系统重复做一套脆弱的 AI 执行治理。

Identity

RunPrincipal 全链路传播

start、chat、resume 都支持 principal overload。subject、clientId、delegation chain 和 roles 可进入 ExecutionContext、policy、人审任务、挂起快照和 Trace。

Security

Spring endpoint security bridge

控制面端点支持 shared-secret 开发模式和 OAuth2 resource-server 模式。Runtime、Console、Studio、Human Review、Events 各自配置 scope。

Policy

角色门禁与额度策略

security.action-roles[] 为 action 配置 any-of 角色；治理模块提供金额提取、hard-limit、review-limit、脱敏和风险审批链路由。

Human Review

高风险 Action 挂起

requiresHumanReview 的 Action 创建审批任务并保存快照。审批通过后同一 runId 恢复；拒绝后进入补偿路径，不继续猜测。

Event Wait

等待外部系统，而不是阻塞线程

ActionResult.waiting(...) 将运行挂起为 SUSPENDED_WAITING_EVENT。EventApplier 折叠回调数据，运行继续使用同一 Trace 序列。

Audit

Trace、hash chain 与只读 Console

Trace 事件按 runId 形成证据链，Console 只读查询和 CSV/JSONL 导出面向审计归档，不执行、恢复、审批或补偿运行。

从运行能力到推广标准，ActionGraph 已进入工程化阶段。

当前版本将 Goal 生产、解释质量、端点安全、运行观测和 API 兼容性纳入工程体系。团队不再只验证单点功能，而是用一套可审计、可评测、可升级的标准化机制推动落地。

DX4

配置化 Goal 与 bundle 指纹

actiongraph.goals.definitions[] 和 bundle import 将外部 Goal 声明编译为 GoalDefinition。action graph fingerprint 与 bundle fingerprint 用于上线一致性检查。

Goal Studio

非生产环境起草 Goal

Studio 只生成 Goal declarations，不生成计划或 Action 顺序。它会自动验证可达性、尝试修复诊断、计算风险 profile，并导出 approved YAML bundle。

Interpretation Quality

解释器评测飞轮

JSONL golden set 评估 goal type、parameters 和 clarification accuracy；生产采样保存 masked input、outcome、fallback 和 parse failure，用于持续标注。

Observability

Micrometer 与 OpenTelemetry GenAI

ObservationSink 支持 Spring 自动装配。OTel sink 写入 GenAI semantic convention 属性，但不记录 prompts、模型输出、request body、tokens 或 Blackboard 原值。

Compatibility

公共 API 快照与稳定性标注

public/protected surface 由 snapshot 和 japicmp 守护。@Experimental 与 @Internal 明确区分试点 API 和非应用代码依赖面。

Control Plane

Java 8 客户端与组件目录

老系统可只引入 actiongraph-control-plane-api，通过 HTTP 调用 Runtime、Catalog、Human Review 和 Console，不把现代运行时 jar 放进旧 JVM。

模块生态围绕部署边界设计。

现代 Java 服务、Spring Boot 应用、旧 Java 8 系统、审批门户、审计控制台和部署检查工具都能只引入自己需要的 artifact。

platform

actiongraph-bom

统一 ActionGraph 各组件版本，适合 Spring Boot 服务和多模块业务仓库。

core

actiongraph-core

Action、GOAP Planner、Executor、GoalCatalog、Blackboard、Trace、RunPrincipal、Observation SPI。

spring

spring-boot-starter

注解扫描、schema seeding、JDBC、治理、人审、控制面、Console、Studio 和安全桥自动装配。

java8-client

control-plane-api

Java 8 HTTP clients、组件目录、控制层错误契约、安全 GET retry 和 shared-secret verifier。

persistence

persistence-jdbc

Trace、SuspendedRun、Memory、人审任务、解释采样和 Console read model 的 JDBC 仓储。

review

human-review

审批任务仓储、审批链、回调处理、任务查询和决策服务，可独立于 Spring 复用。

governance

governance

脱敏、额度规则、权限策略、租户校验、风险审批链路由和 review attributes。

ops

actiongraph-console

只读运行查询、Trace 时间线、审计链验证、CSV/JSONL 导出和 Goal Studio 服务。

llm

llm-deepseek

GoalCatalog prompt、DeepSeek/OpenAI-compatible client、解释解析、fallback 和 golden-set evals。
